Cooperative games¶

nlp-shap explains model behaviour by treating input units (tokens, segments, etc.) as players in a cooperative game. Each coalition \(S \subseteq N\) of players yields a payoff \(v(S)\) from a characteristic function \(v\) defined by a value function and model inference.

Characteristic function¶

A cooperative game is given by:

  • a finite player set \(N = \{1, \dots, n\}\)

  • a function \(v : 2^N \to \mathbb{R}\) with \(v(\emptyset)\) as the empty-coalition reference

In explainability, \(v(S)\) is the utility of the model output when only the players in \(S\) are present (others are masked according to an absence policy). The grand coalition \(v(N)\) is the unmasked input.

Marginal contributions¶

For player \(i\) and coalition \(S \subseteq N \setminus \{i\}\) the marginal contribution is:

\[\Delta_i(S) = v(S \cup \{i\}) - v(S)\]

Attribution methods differ in how they weight and aggregate marginal contributions across coalitions — see Estimands: Shapley vs Banzhaf.

Phase 0 scope¶

Version 0.1.1 ships estimand aggregators only. Coalition sampling, masking, backends, and the full explain pipeline arrive in later phases. The aggregators assume you supply coalition masks and aligned payoffs \(v(S)\).
